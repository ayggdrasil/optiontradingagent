P&L Tracking

Position P&L Fields

How request_key → P&L is bridged
Pass saved request_keys[] to callput_portfolio_summary. Internally: openPositionRequests(key).optionTokenId maps each key to the ERC-1155 token. amountIn (index [5]) is the on-chain cost basis. All 7 P&L fields below are computed automatically.
Field Type Description
entry_cost_usd USD On-chain cost basis from openPositionRequests(key).amountIn. Authoritative — never estimated.
current_value_usd USD Current mark-price spread value (mid fair value). Theoretical — not what you'd receive on close.
unrealized_pnl_usd USD current_value_usd − entry_cost_usd. Mark-based. Can be misleading in wide bid/ask markets.
unrealized_pnl_pct % Unrealized P&L as % of entry cost. Mark-based.
close_bid_value_usd USD Bid-based close estimate: what the keeper would fill if you closed now. Can be negative for deeply underwater positions.
close_pnl_est_usd USD close_bid_value_usd − entry_cost_usd. Use this for all exit decisions — it's the realistic P&L.
close_pnl_est_pct % Close P&L as % of entry cost. Profit-taking rule: consider closing when > 50%.
◈
Profit-taking rule: Consider closing when close_pnl_est_pct > 50. Always use close_pnl_est_usd (bid-based), not unrealized_pnl_usd (mark-based), for exit decisions. Use callput_close_position() pre-expiry · callput_settle_position() post-expiry.
Realized P&L (post-settlement): Call callput_get_settled_pnl() to get amount_out_usd per position from on-chain SettlePosition events. Net realized P&L = amount_out_usd − entry_cost_usd. Default lookback: ~50k blocks (~1 day on Base). Set from_block for older positions.
Strategy Mechanics

Spread Payoff at Expiry

All 4 strategies have capped profit and capped loss. Buy spreads pay premium upfront for directional exposure — max loss is the premium paid, max profit is the spread width minus premium. Sell spreads collect premium as collateral credit — max profit is the credit received, max loss is the spread width minus credit. The B/E marker shows the breakeven price at expiry.

bullish BuyCallSpread
Max profit = spread width − premium Max loss = premium paid
bearish BuyPutSpread
Max profit = spread width − premium Max loss = premium paid
neutral-bearish SellCallSpread
Max profit = credit received Max loss = spread width − credit
neutral-bullish SellPutSpread
Max profit = credit received Max loss = spread width − credit

Operator Setup Guide

01
Clone & Build
# Clone the repository
git clone https://github.com/ayggdrasil/callput-option-agent callput-option-agent
cd callput-lite-mcp-skill-standalone

npm install
npm run build

# Verify — stdio MCP server, connect via MCP client
node build/index.js --help
02
MCP Server Configuration
Add to your MCP config: ~/.config/mcp/config.json, Claude Desktop claude_desktop_config.json, or OpenClaw/Bankr server config.
{
  "mcpServers": {
    "callput": {
      "command": "node",
      "args": ["/absolute/path/to/callput-option-agent/build/index.js"],
      "env": {
        "RPC_URL": "https://mainnet.base.org"
      }
    }
  }
}
No private key required — MCP only builds unsigned transactions. The agent signs with its own key.
VariableRequiredDefaultDescription
RPC_URL optional mainnet.base.org Base Mainnet RPC endpoint (read-only).
03
Inject Agent System Prompt
Inject SKILL.md as the agent's system prompt or skill document. Contains all trading rules, unsigned-tx flow, and IV-based strategy selection.
# Key rules:
Rule 01: Spread-only — no single-leg execution
Rule 06: MCP never holds private key — agent signs externally
Rule 07: If usdc_approval.sufficient == false → send approve_tx first
Rule 08: Persist every request_key from get_request_key_from_tx
Rule 09: If request_keys lost → use list_positions_by_wallet
Rule 10: High IV (ETH >80%, BTC >70%) → favor sell spreads
04
Register SKILL.md (Optional, Recommended)
If your agent runtime supports skill/persona documents, register SKILL.md as the agent skill. Provides fast-path workflow, strike selection guidance, complete P&L tracking pattern, and bias-to-strategy mapping.
05
First Session Checklist
  • callput_portfolio_summary({ address: "0x..." }) — check balance and open positions
  • callput_scan_spreads({ underlying_asset: "ETH", bias: "bullish" }) — get ranked candidates + atm_iv
  • callput_execute_spread({ ...rank1, from_address: "0x...", size: 1 }) — get unsigned_tx + usdc_approval
  • [if usdc_approval.sufficient == false] sign + broadcast approve_tx first
  • sign + broadcast unsigned_tx → get tx_hash
  • callput_get_request_key_from_tx({ tx_hash }) — parse + persist request_key
  • callput_check_request_status({ request_key, is_open: true }) — poll every 30s until executed
Protocol

About Callput.app

◈
Spread-Only Options
Callput is a spread-only options protocol on Base. No single-leg exposure. USDC settlement only. Positions are ERC-1155 tokens.
⬡
Keeper-Based Execution
Orders are filled asynchronously by a keeper relay. Poll check_request_status after broadcast. Keeper fills within ~30s typically.
◇
On-Chain Contracts
PositionManager + SettleManager on Base Mainnet.
PositionManager:
0x83B04701B227B045CBBAF921377137fF595a54af
▲
Supported Assets
BTC and ETH only. 4 spread strategies: BuyCallSpread, SellCallSpread, BuyPutSpread, SellPutSpread. All collateral and settlement in USDC.
