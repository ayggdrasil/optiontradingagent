// 01 — ARCHITECTURE

HOW IT WORKS

01
AGENT LAYER
OpenClaw, Bankr, or any MCP-compatible AI agent interprets market context, selects bias, and issues MCP tool calls. The agent maintains minimal session state: asset, bias, expiry code, leg IDs, and the list of request_keys for P&L tracking.
OpenClaw / Bankr
02
SKILL LAYER
SKILL.md and EXTERNAL_AGENT_PROMPT.md encode the complete trading rulebook — 14 hard rules covering execution safety, position sizing, skip conditions, and P&L discipline. The agent reads these at session start via callput_bootstrap.
SKILL.md
03
MCP SERVER
Node.js stdio server built with @modelcontextprotocol/sdk. Exposes 10 tools with Zod-validated schemas over JSON-RPC. Bridges agent intent to on-chain actions with full type safety and structured error responses.
MCP stdio · TypeScript
04
CORE LOGIC
scanSpreads() uses ATM-anchored spread generation: long leg fixed at nearest-to-spot strike, varying width 1/2/3 strikes across 2 expiries — max 6 candidates before ranking. getPortfolioSummary() enriches positions with live spread mark values and resolves on-chain cost basis from openPositionRequests(key).amountIn.
core.ts · ethers v6
05
BLOCKCHAIN LAYER
ethers.js v6 signs and broadcasts transactions to Base Mainnet RPC. PositionManager contract emits GenerateRequestKey events. An async keeper relay picks up signed requests and fills orders — agents poll checkRequestStatus until executed or cancelled.
Base Mainnet · Chain 8453
06
EXTERNAL SERVICES
S3 market data feed delivers real-time option chains with a 5-second in-memory cache to minimize redundant fetches. USDC allowance is auto-managed — ensureAllowance() pre-approves 2× the required amount, eliminating manual approval steps for agents.
S3 · USDC ERC-20

// 02 — MCP TOOLS

10 TOOLS, ZERO NOISE

callput_scan_spreads
NEW
Primary market entry. Bias-driven, ATM-anchored scan. Returns ≤5 ranked, ready-to-execute spread candidates. No chain parsing — pass results directly to execute_spread.
IN · asset · bias · max_results?
OUT · ranked candidates · cost_pct_of_max · long/short_leg_id
callput_portfolio_summary
UPDATED
USDC balance + enriched positions + live P&L. Pass saved request_keys to unlock per-position cost basis, unrealized P&L (mark), and bid-based close estimate (tradeable).
IN · address? · request_keys[]
OUT · usdc_balance · urgent_count · entry_cost_usd · unrealized_pnl_usd · close_bid_value_usd · close_pnl_est_usd · close_pnl_est_pct
callput_execute_spread
CORE
Execute spread or dry-run tx payload. USDC auto-approved at 2× required. Returns request_key — persist immediately. dry_run=true by default; requires explicit auth for live.
IN · strategy · long/short_leg_id · size · dry_run
OUT · request_key · tx_hash · estimated_cost_usd
callput_validate_spread
CORE
Pre-execution validation. Verifies leg compatibility and ordering: call spreads long lower / short higher; put spreads long higher / short lower. Use when not on scan path.
IN · strategy · long_leg_id · short_leg_id
OUT · valid · errors[] · cost_estimate_usd
callput_check_request_status
CORE
Poll keeper after broadcast. Call every 30s until status is executed or cancelled. Maximum 3-minute polling window. Required after every live execute_spread call.
IN · request_key · is_open
OUT · status · fill_price · size_out · execution_block
callput_get_positions
CORE
Fetch active option positions for a wallet address. Returns raw positions with option_token_id — required for close and settle operations downstream.
IN · address?
OUT · positions[] · days_to_expiry · mark_price
callput_close_position
CORE
Pre-expiry exit for open positions. Use when days_to_expiry < 1, or when taking profit at close_pnl_est_pct > 50 (bid-based 50% gain — use this, not mark price). Supports dry-run.
IN · asset · option_token_id · size · dry_run
OUT · request_key · tx_hash
callput_settle_position
CORE
Post-expiry settlement only. Distinct from close — use ONLY for expired positions to realize the spread payout on-chain via SettleManager contract.
IN · asset · option_token_id · dry_run
OUT · settlement_amount_usd · tx_hash
callput_get_option_chains
CORE
Raw option chain data from S3 market feed. Use scan_spreads as the primary entry point — only reach for this for advanced analysis beyond what scan_spreads provides.
IN · asset · option_type? · expiry_date? · max_strikes?
OUT · option_chains[] · spot_price · timestamp
callput_bootstrap
CORE
Session initialization. Returns minimal workflow, 14 strict rules, all tool names, and required session state fields. Call first to orient any fresh agent session.
IN · (none)
OUT · rules[] · flow[] · tools[] · minimal_state[]

// 03 — TRADE FLOW

THE WORKFLOW

1
Pre-Trade Portfolio Check
Before every new position: check USDC balance and open positions. Skip if usdc_balance < 2× spread cost. If urgent_count > 0, manage expiring positions before opening new ones.
callput_portfolio_summary({ request_keys: state.request_keys })
// → usdc_balance, positions[], urgent_count, total_pnl_usd
2
Bias-Driven Market Scan
Scan by asset and directional bias. Gets ≤5 ATM-anchored spreads ranked by cost_pct_of_max ascending. Skip any candidate where cost_pct_of_max > 40% — poor risk/reward. Prefer rank 1.
callput_scan_spreads({ underlying_asset: "ETH", bias: "bullish" })
// → candidates ranked · pick rank 1 if cost_pct_of_max ≤ 40%
3
Dry-Run Confirmation
Confirm the transaction payload before committing. Inspect legs, estimated cost, and size. The safety gate — never skip this step even if the agent has prior authorization.
callput_execute_spread({ ...rank1, size: 1, dry_run: true })
// → tx payload preview · no on-chain state change
4
Live Execution
Real on-chain execution. Requires explicit user authorization in this session. CALLPUT_PRIVATE_KEY must be set in MCP env. USDC is auto-approved at 2× required — no manual step needed.
callput_execute_spread({ ...rank1, size: 1, dry_run: false })
// → request_key ← SAVE THIS IMMEDIATELY
5
Poll the Keeper
After broadcast, the keeper processes the signed request asynchronously. Poll every 30 seconds, up to 3 minutes, until status reaches executed or cancelled.
callput_check_request_status({ request_key, is_open: true })
// repeat every 30s · max 6 attempts · stop on executed | cancelled
6
Persist request_key for P&L
Push the request_key into session state. On the next portfolio_summary call, the server resolves openPositionRequests(key).optionTokenId to bridge it to the live ERC-1155 position — unlocking entry_cost_usd, unrealized_pnl_usd (mark), and close_pnl_est_usd (bid-based tradeable). Consider closing when close_pnl_est_pct > 50.
state.request_keys.push(result.request_key)
// unlocks: entry_cost_usd · unrealized_pnl_usd · close_pnl_est_pct

// 04 — AGENT COMMANDS

GIVING ORDERS

Agents understand natural-language directives and map them to the MCP workflow automatically. Be explicit about asset, bias, and execution authorization. The skill enforces all safety rules — your job is to supply intent, not mechanics.

HARD RULES (AUTO-ENFORCED)

  • 01Spread-only. No single-leg execution ever.
  • 02Always check portfolio_summary before opening a new position.
  • 03Use scan_spreads — not raw chains — as the primary entry point.
  • 04Validate before execute if not on the scan path.
  • 05dry_run=true unless you explicitly say "execute for real" or "I authorize live execution."
  • 06Skip if cost_pct_of_max > 40% — poor risk/reward ratio.
  • 07Skip if days_to_expiry < 0.25 (under 6 hours to expiry).
  • 08Manage urgent positions first if urgent_count > 0.
  • 09Save every request_key returned by execute_spread.
  • 10CALLPUT_PRIVATE_KEY never appears in any output or log.

Per-Position P&L Fields

Pass request_keys[] to callput_portfolio_summary to unlock full per-position cost basis and P&L. The server bridges each key to its ERC-1155 token via openPositionRequests(key).optionTokenId on-chain.

FIELDDESCRIPTION
entry_cost_usdOn-chain cost basis from openPositionRequests(key).amountIn
current_value_usdLive mark-price spread value (fair mid)
unrealized_pnl_usdcurrent_value − entry_cost (mark-based, theoretical)
unrealized_pnl_pctUnrealized P&L as % of entry
close_bid_value_usdBid-based close estimate (naked.bid − pair.ask) — tradeable
close_pnl_est_usdclose_bid_value − entry_cost (realistic exit P&L)
close_pnl_est_pctExit P&L as % of entry — use for profit-taking decisions
Profit-taking rule: consider closing when close_pnl_est_pct > 50.
Use close_pnl_est_usd for real decisions — not unrealized_pnl_usd. The bid-based estimate reflects what you'd actually receive on close, not the theoretical mid.

// 06 — CALLPUT.APP

THE PROTOCOL

BASE MAINNET · CHAIN ID 8453

Callput is a spread-only options protocol built on Base Mainnet. Every position is a defined-risk two-leg spread — maximum loss is known at entry, no unlimited downside. USDC is the settlement currency throughout.

The protocol uses an async keeper architecture: agents broadcast signed transactions via PositionManager, the keeper relays them on-chain, and order status is tracked via GenerateRequestKey events. Settlement of expired positions goes through a separate SettleManager contract.

This MCP skill — callput-lite-agent-mcp — is the canonical interface for AI agents trading on Callput. Built in TypeScript with ethers.js v6 and the @modelcontextprotocol/sdk.
