거래 실행 플로우 (권장 순서)
→
STEP 1
portfolio_summary
USDC 잔고 + 현재 포지션 확인. urgent_count > 0이면 만기 관리 먼저.
→
STEP 2
scan_spreads
asset + bias 입력 → 최대 5개 순위별 후보. rank 1 = 최고 가성비.
→
STEP 3
execute_spread
(dry_run=true)
rank 1 후보로 tx payload 확인. 실제 서명 없음.
→
STEP 4
execute_spread
(dry_run=false)
유저 명시 승인 후 실행. USDC 자동 approve 포함.
→
STEP 5
check_request_status
30초마다 폴링. executed / cancelled 확인.
→
STEP 6
request_key 저장
에이전트 상태에 반드시 저장. P&L 추적에 필수.
STEP 7
close / settle
days_to_expiry < 1 → close. 만기 후 → settle.

NEW

OUTrules[], tools[], minimal_state[]

callput_validate_spread

callput_execute_spread
MCP
스프레드 실행. dry_run=true(기본)는 tx payload만 반환. 실행 시 USDC 자동 approve + keeper 대기.

MCP

절대 위반 불가 규칙
CALLPUT_PRIVATE_KEY는 MCP 환경변수에만 존재. 어떤 출력/로그에도 노출 금지.
Frontend(ARCHITECTURE.html, index.html)는 키 미보관. 실행 권한 없음.
Single-leg 직접 실행 금지. 항상 스프레드 쌍으로만 거래.
dry_run=true가 기본값. 유저 명시 승인 없이 dry_run=false 전환 금지.
main/master에 force push 금지. 모든 배포는 PR 경유.

scan_spreads 내부 동작 — 조합 폭발 억제 원리
입력
asset = ETH
bias = bearish
→
자동 결정
optionType = Put
strategy = BuyPutSpread
expiry = 가장 빠른 >6h
longLeg = ATM strike
→
3가지 width 생성
narrow: ATM / ATM−1
medium: ATM / ATM−2
wide: ATM / ATM−3
→
출력 (최대 5개)
rank 1: cost_pct 18%
rank 2: cost_pct 24%
rank 3: cost_pct 31%
long_leg_id + short_leg_id
바로 execute에 사용 가능
기존 방식: 체인 조회 → 수백 개 조합 에이전트가 직접 파싱 → 수십 turn 소요
신규 방식: bias 1개 입력 → 3개 후보 → rank 1 바로 실행 → 2 turn 완료
